# Бизнес-требования
## Приложение: Zori — голосовой тренер для продавцов

---

## 1. Главная страница

### 1.1 Хедер
- Название приложения: **Zori** (текстом, без логотипа)
- Фиксированный, отображается всегда поверх контента
- Для роли `admin` — кнопка перехода в админ-панель (открывается в новом окне)

### 1.2 Кнопка управления сессией
- Центральный элемент страницы
- **Состояние "Старт":** пиктограмма треугольник (play), цвет зелёный
- **Состояние "Активна":** пиктограмма квадрат (stop), цвет красный
- При нажатии "плей" — запускается двусторонний голосовой режим
- При нажатии "стоп" — голосовой режим останавливается
- Кнопка фиксированная — всегда видна на экране, не уезжает при скролле транскрипта

---

## 2. Голосовой режим (активная сессия)

### 2.1 Пульсация
- Два независимых анимированных индикатора пульсации:
  - **Слева** — пульсация пользователя, цвет: `#4A90D9` (синий)
  - **Справа** — пульсация AI, цвет: `#F5A623` (оранжевый)
- Пульсация активна только у того, кто говорит в данный момент
- У молчащего — индикатор статичен

### 2.2 Транскрипт
- Отображается в виде диалога, стилизованного под мессенджер (по аналогии с Telegram)
- Реплики пользователя — справа, реплики AI — слева
- Текст появляется в реальном времени (слово за словом)
- После нажатия "стоп" транскрипт остаётся на экране
- При повторном нажатии "плей" новые реплики добавляются продолжением той же беседы

---

## 3. Тесты

**Тест** — это сценарий диалога с заданной персоной. Перед началом сессии пользователь выбирает тест, который определяет роль и поведение AI.

### 3.1 Выбор теста

- Перед нажатием «плей» отображается выпадающий список (select) со всеми активными тестами
- Первый пункт списка — плейсхолдер «Выберите тест» (не является тестом)
- Только после выбора теста кнопка «плей» становится доступной
- Если тест не выбран — кнопка «плей» неактивна
- Выбор теста сохраняется до конца сессии; при повторном нажатии «плей» тест остаётся тем же

### 3.2 Структура теста

| Поле | Описание |
|---|---|
| Название | Короткое имя, например «Скептичный закупщик» |
| Описание | 1–2 строки — кто это и в какой ситуации |
| Инструкция для AI | Полный промпт: роль, характер, контекст, возражения |
| Статус | Активен / отключён |

### 3.3 Управление тестами

- Тесты создаёт, редактирует и отключает администратор через вкладку «Тесты» в админ-панели
- Активные тесты доступны всем пользователям
- Отключённый тест не отображается у пользователей, но не удаляется из базы

---

## 4. Логи бесед

- Все реплики сохраняются автоматически в SQLite
- Каждая сессия однозначно привязана к конкретному тесту
- Каждая сессия фиксирует время начала и окончания
- Каждая реплика фиксирует: автора (пользователь / AI), текст, время
- Реплики привязаны к конкретному пользователю через сессию
- Галлюцинации и прерванные реплики AI не сохраняются

---

## 5. Авторизация

### 5.1 Общие принципы
- Приложение поддерживает несколько пользователей (до нескольких сотен)
- Каждый пользователь видит только свои логи
- Аккаунты создаются и управляются администратором через интерфейс

### 5.2 Вход
- Авторизация по логину и паролю
- Пароль хранится в виде bcrypt-хэша, никогда в открытом виде
- После входа пользователь получает JWT-токен

### 5.3 Токены
- Время жизни токена: **7 дней**
- Токены хранятся в БД (хэш, не сам токен)
- При выходе токен удаляется из БД
- При блокировке пользователя все его токены удаляются немедленно
- Истёкшие токены очищаются автоматически

## 6. Ролевая модель

### 6.1 Роли

| Роль | Возможности |
|---|---|
| `admin` | Управление пользователями, просмотр логов всех пользователей |
| `user` | Ведение сессий, просмотр только своих логов |

### 6.2 Интерфейс администратора
- Доступ через кнопку в хедере (только для роли `admin`), открывается в новом окне
- Одна страница с тремя вкладками: **Пользователи**, **Тесты** и **Логи**
- При переходе между вкладками с незавершённой операцией — запрашивается подтверждение:
  - "Да" — завершить операцию и перейти
  - "Нет" — перейти без сохранения

### 6.3 Вкладка "Пользователи"
- Таблица со списком всех пользователей
- У каждой строки кнопки действий:
  - Создать пользователя (логин, пароль, роль)
  - Сбросить пароль
  - Заблокировать / разблокировать
  - Удалить

### 6.4 Вкладка "Тесты"
- Таблица со списком всех тестов (активные и отключённые)
- У каждой строки кнопки действий:
  - Создать тест (название, описание, инструкция для AI)
  - Редактировать
  - Включить / отключить

### 6.5 Вкладка "Логи"
- Список всех пользователей
- При выборе пользователя — отображается список его сессий (дата, время)
- При выборе сессии — отображается полный транскрипт беседы (с указанием теста)

---

## 7. Общие требования

### 7.1 Цветовая схема
- Светлая тема
- Акцентные цвета: зелёный (старт), красный (стоп), синий (пользователь), оранжевый (AI)

### 7.2 Адаптивность
- Полная мобильная адаптация (responsive)
- Приоритет: корректное отображение на экранах от 375px (iPhone SE) и выше
- На десктопе контент центрируется, максимальная ширина ~800px
