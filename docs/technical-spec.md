## 3. Схема взаимодействия компонентов

```
[Браузер]
    │
    │  1. GET /api/session/token  (с JWT в заголовке)
    ▼
[Node.js сервер]
    │
    │  2. POST https://api.openai.com/v1/realtime/sessions
    │     (OPENAI_API_KEY хранится только здесь)
    ▼
[OpenAI REST API]
    │
    │  3. ephemeral token (живёт 60 сек)
    ▼
[Node.js сервер]
    │
    │  4. отдаёт token фронтенду
    ▼
[Браузер]
    │
    │  5. WebRTC соединение напрямую с OpenAI
    ▼
[OpenAI Realtime API]
    │  ← аудио пользователя (стримингом)
    │  → аудио AI + транскрипт (стримингом)
```

**Важно:** API ключ хранится только на сервере. Фронтенд работает с временным ephemeral token.

## 5. Авторизация

### 5.1 Вход
- `POST /api/auth/login` — принимает `{ username, password }`, проверяет bcrypt, возвращает JWT
- JWT содержит: `{ userId, username, role }`
- Время жизни токена: **7 дней**
- Хэш токена сохраняется в таблице `tokens`

### 5.2 Выход
- `POST /api/auth/logout` — удаляет хэш токена из таблицы `tokens`

### 5.3 Middleware
- Все `/api/session/*` и `/api/admin/*` маршруты защищены JWT-middleware
- Middleware проверяет: подпись токена, наличие хэша в таблице `tokens`, статус пользователя (`active`)
- Если пользователь заблокирован — все его токены удаляются немедленно

### 5.4 Очистка токенов
- Истёкшие токены очищаются при каждом запуске сервера и по расписанию (раз в сутки)

## 6. Голосовой модуль

### 6.1 Общие принципы
- Всё взаимодействие — стриминговое, без задержек на буферизацию
- Язык: только русский
- Режим: двусторонний голосовой диалог без нажатия кнопок в процессе общения

### 6.2 Голосовой ввод (пользователь → система)
- Захват микрофона начинается сразу после нажатия кнопки "плей"
- VAD (Voice Activity Detection) — встроенный в OpenAI Realtime API
- Модель распознавания речи: `gpt-4o-transcribe`, язык: `ru`
- Речь стримингом передаётся в OpenAI для распознавания
- Распознанный текст отображается в транскрипте в реальном времени (слово за словом)

### 6.3 Голосовой вывод (AI → пользователь)
- AI начинает отвечать сразу после окончания реплики пользователя
- Ответ синтезируется и воспроизводится стримингом
- Параметры голоса AI: мужской, ~40 лет, низкий тенор

### 6.4 Обработка перебивания (interruption)
- Если пользователь начинает говорить во время реплики AI:
  - Воспроизведение голоса AI останавливается немедленно
  - Генерация ответа AI прерывается
  - Система переходит в режим прослушивания пользователя

### 6.5 Контент диалога
- AI общается на любые темы без заданной роли или сценария
- Диалог не сбрасывается при паузе — при повторном "плей" контекст сохраняется

### 6.6 Модель распознавания голоса (вход)

#### Транскрипция
| Параметр | Описание | Текущее значение |
|---|---|---|
| `input_audio_transcription.model` | Модель распознавания речи | `gpt-4o-transcribe` |
| `input_audio_transcription.language` | Язык распознавания | `ru` |

#### VAD — детекция голосовой активности
| Параметр | Описание | Текущее значение |
|---|---|---|
| `turn_detection.type` | Тип детекции | `server_vad` |
| `turn_detection.threshold` | Порог громкости (0–1): выше = только чёткая речь | `0.9` |
| `turn_detection.silence_duration_ms` | Пауза тишины (мс) перед отправкой реплики | `1000` |
| `turn_detection.prefix_padding_ms` | Захват аудио (мс) до начала речи | `300` |

---

### 6.7 Модель генерации голоса (выход)

| Параметр | Описание | Текущее значение |
|---|---|---|
| `model` | Модель диалога и синтеза речи | `gpt-realtime` |
| `voice` | Голос AI | `cedar` |
| `instructions` | Системный промпт: роль, язык, стиль | берётся из поля `instructions` выбранного теста |
| `temperature` | Креативность ответов (0–1) | `0.7` |
| `max_response_output_tokens` | Лимит токенов на один ответ | `250` |
| Контекст | Максимальный размер диалога | 32k токенов |
| Стоимость | Примерная стоимость | ~$0.048/мин (вход) + ~$0.19/мин (выход) |

## 7. API маршруты

### Авторизация
| Метод | Путь | Описание | Доступ |
|---|---|---|---|
| POST | `/api/auth/login` | Вход, возврат JWT | публичный |
| POST | `/api/auth/logout` | Выход, удаление токена | авторизованный |

### Тесты (пользователи)
| Метод | Путь | Описание | Доступ |
|---|---|---|---|
| GET | `/api/tests` | Список активных тестов (без поля `instructions`) | `user`, `admin` |

### Сессии
| Метод | Путь | Описание | Доступ |
|---|---|---|---|
| POST | `/api/session/token` | Получить ephemeral token + создать сессию (body: `{ testId }`) | `user`, `admin` |
| POST | `/api/session/end` | Завершить сессию | `user`, `admin` |
| POST | `/api/session/message` | Сохранить реплику | `user`, `admin` |
